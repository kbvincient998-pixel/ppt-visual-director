# PowerPoint 执行与 QA

## 路由

先运行 `scripts/audit_pptx.py`。

- 没有视频、动画、宏或 OLE：使用 `Presentations` Skill 和 `@oai/artifact-tool` 处理。
- 存在高风险对象：保留原 PPTX 结构，在副本中通过 PowerPoint 界面完成必要编辑。
- 只有普通转场时，先确认是否必须保留；必须保留则走 PowerPoint 界面，不要求保留时才允许文件级编辑。
- 不把媒体复杂文件整体导入再导出，除非兼容性往返测试证明对象完整保留。
- 审计引用字体和嵌入字体。字体未嵌入时，批量处理前确认当前电脑已安装关键字体。

## 样页阶段

选择 3–5 个不同角色页面，不只选择最容易美化的页面。样页必须暴露整稿的真实难点：

- 品牌主视觉。
- 产品锁定与 IMAGE2IMAGE。
- 高密度策略或洞察。
- 真实截图或达人证据。
- 机制、路线或视频页面。

确认样页后，把视觉规则写入本次任务的设计说明，再批量执行。

## 大文件降级路线

- 整份 PPTX 渲染内存不足时，不降低为“只看几页”。按章节或固定页段分批渲染后合并检查。
- PowerPoint 导出 PDF 失败时，优先使用项目中已有的同版本 PDF；先核对页数、封面和至少 3 个跨章节代表页。
- 同版本 PDF 只代表静态画面。媒体、动画、转场、嵌入对象和可编辑性仍以 PPTX 审计、媒体比对和 PowerPoint 抽检为准。
- 不把低分辨率联系表当成最终逐页 QA；联系表只检查全稿节奏。

## 编辑原则

- 保留标题、正文、关键数字、表格、图表和主要图片框的可编辑性。
- 氛围背景可以合成，但不得覆盖真实证据或把整页扁平化。
- 文字过多时先改写或换版式，不靠持续缩小字号解决。
- 不随意删除母版、页码、Logo、脚注和来源。
- 对视频页优先改变周围文字和静态元素，不重建视频对象。

## QA 顺序

1. 运行布局或溢出检查。
2. 渲染全部页面。
3. 用总览图检查全稿节奏和风格一致性。
4. 逐页全尺寸检查文字、图片、裁切、层级和品牌准确性。
5. 对每个产品图执行原图/成图对照。
6. 对媒体复杂文件运行 `scripts/compare_pptx_media.py`。
7. 在 PowerPoint 中抽检可编辑文字、图片替换、图表和视频播放。

## 不通过条件

- 包装、Logo、色号、人物身份或真实证据发生变化。
- 标题换行、文字溢出、对象越界或意外遮挡。
- 视频、动画、宏、嵌入对象或页面数量丢失。
- 图片与文案只在颜色上相似，语义无关。
- 证据页被处理成失真的装饰拼贴。
- 成稿覆盖源文件，或最终使用的生成图片未保存到项目中。
